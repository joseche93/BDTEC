  • Qué es una base de datos federada?

    El avance espectacular de las comunicaciones y la creciente necesidad de cooperar con otras entidades independientes, obligan a replantear los conceptos fundamentales de las bases de datos, dirigiéndose de forma directa a la reciente tendencia de crear software para tener acceso a varias bases de datos autónomas preexistentes almacenadas en SGBD heterogéneos.

    Pero el acceso a varias bases de datos como si de una sola se tratara viene acompañado de problemas como la integración, seguridad, control, etc. Para poner solución a estos problemas y ofrecer un adecuado acceso a varias bases de datos se han desarrollado nuevos esquemas como los Sistemas Gestores de Bases de Datos Federadas (SGBDF).

    Un sistema de bases de datos federadas es una colección de sistemas de bases de datos cooperativos y autónomos. En un sistema federado los usuarios tienen acceso a los datos, de los distintos sistemas, a través de una interfaz común sin embargo, no existe un esquema global que describa a todos los datos de las distintas bases de datos, en su lugar hay varios esquemas unificados, cada uno describiendo porciones de bases de datos y archivos para el uso de cierta clase de usuarios.

    Las bases de datos federadas son vistas unificadas de bases de datos independientes aparentan ser una sola base de datos, pero son una colección de sistemas de bases de datos independientes, cooperativos, heterogéneos, que son autónomos y que permiten compartir todos o algunos de sus datos. Una BDF aparenta ser una BD normal y corriente, pero no tiene existencia física, es una vista lógica.

  • Qué es un sistema multidatabase?

    Un sistema multibase de datos (SMulBD) soporta operaciones en múltiples sistemas de base de datos componentes (SBDC). Cada SBDC es manejado por un sistema manejador de base de datos (SMBD). Un SBDC en un SMulBD puede ser centralizado o distribuido y puede residir en la misma computadora o en múltiples computadoras conectadas por un subsistema de comunicación. Un SMulBD es llamado homogéneo si todos los SMBD componentes son iguales; si son diferentes entonces es llamado un SMulBD heterogéneo.

    Un SMulBD puede ser clasificado en dos tipos basados en la autonomía de la SBDCs: sistemas de base de datos no-federada y sistemas de base de datos federada.

  • Cluster Databases

    Clustering, en el contexto de las bases de datos, se refiere a la capacidad de varios servidores o instancias para conectarse a una sola base de datos. Una instancia es la colección de memoria y los procesos que interactúan con una base de datos Clustering ofrece dos ventajas importantes, especialmente en entornos de bases de datos de gran volumen:

    La tolerancia a fallos: esto implica que hay más de un servidor o instancia para que los usuarios se conecten, es decir, la agrupación ofrece una alternativa en caso de fallo de un servidor individual.

    El balanceo de carga: la característica de clustering generalmente permite que los usuarios sean asignados automáticamente al servidor con la menor carga.

  • SAP HANA distribuited in memory database system: Transaction, session, and metadata management.

    La base de datos SAP HANA es el componente central de la hoja de ruta de SAP HANA para apoyar de manera eficiente todos los procesos de negocio de SAP y no SAP desde una perspectiva de gestión de datos. En frente a la arquitectura tradicional de un sistema de base de datos, la base de datos SAP HANA tiene un enfoque diferente para proporcionar soporte para una amplia gama de tareas de gestión de datos. Por ejemplo, el sistema está organizado de una manera centrada en la memoria principal para reflejar el cambio dentro de la jerarquía de memoria y para proporcionar un alto rendimiento y sin ninguna interacción de disco lentos.

    El enfoque centrado en la memoria principal, por tanto, pone al revés la clásica arquitectura de paradigma : Mientras que los sistemas centrados en disco tradicionales tratan de adivinar los datos calientes para ser almacenado en caché en la memoria principal, los valores por defecto de aproximación SAP HANA pueden tener todo en la memoria principal.

    Además del rendimiento, la base de datos SAP HANA también apunta a apoyar los procesos de negocio desde una perspectiva holística. Por ejemplo, el sistema puede contener documentos de texto de los productos dentro de un orden, junto con información estructurada del cliente y la información espacial de la ruta de entrega actual. Por último, SAP HANA está posicionado para actuar como una plataforma de consolidación para muchos casos de uso diferentes, desde una perspectiva de la aplicación y desde una perspectiva de gestión de datos.
